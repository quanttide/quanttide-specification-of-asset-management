# 验证配置

定义用于约束"物理现状"与"数字契约"之间对标行为的行为准则集合。

不直接定义资产本身，而是定义"如何判定资产是否合规"的度量衡。

## 字段

- `policies`: 必选。列表类型。有序的治理策略集合，遵循"首位命中（First-match）"原则。

## 策略集

定义不同范围或类型资产的验证规则。

如果一个资产同时符合多个策略，遵循 "首位命中（First-match）"规则，即YAML中排在前面的策略优先级更高。

策略集字段定义如下：

- 选择器`selector`: 必选。Glob 模式，用于从已发现的资产中选定该策略的管辖范围。
- 模式`mode`: 必选。枚举类型（全大写）。定义该范围内的验证严格度。

## 模式

定义验证的严格程度：

- `SCOPED`: 范围模式。基于路径前缀的管辖继承，适用于目录级治理。
- `ATOMIC`: 原子模式。基于精确路径的 1:1 对标，适用于单点条目治理。

## YAML示例

```yaml
validation:
  policies:
    # 策略 1：核心资产（原子治理）
    - selector: "**/configs/*.{yaml,json}"
      mode: ATOMIC

    # 策略 2：素材资源（范围治理）
    - selector: "**/assets/textures/**"
      mode: SCOPED

    # 策略 3：全局兜底（默认范围）
    - selector: "**"
      mode: SCOPED
```
